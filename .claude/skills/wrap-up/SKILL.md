# 뽀숑이 작업 마무리 (wrap-up)

## 왜 필요한가
이 프로젝트는 세션이 바뀌어도 뽀숑이의 최신 상태와 맥락이 끊기지 않아야 한다.
한 세션에서 무게·접종·사료·훈련 같은 사실이 바뀌거나 새로 정해진 게 있으면,
그것을 **여러 저장 위치에 일관되게** 남겨야 다음 세션(그리고 가족·모바일)이 같은 사실을 본다.
이 스킬은 그 마무리 절차를 빠뜨리지 않고 수행하기 위한 체크리스트다.

## 절차

### 1. 이번 세션에서 바뀐 것 추려내기
대화를 되짚어 **영구히 남길 가치가 있는 변화**만 고른다:
- 상태 변화: 몸무게, 접종 차수/날짜, 사료·급여량·시간, 중성화 여부, 훈련 단계
- 새 사실/결정: 알러지·건강 이슈, 운영 방식·선호, 인프라 변경
- 일상 기록: 그날 있었던 일(산책, 배변, 증상 등)

코드/기록에서 이미 자명한 것, 이 대화에서만 의미 있는 잡담은 **남기지 않는다**.

### 2. 상태가 바뀌었으면 "3곳"을 동기화
하나라도 바뀌면 아래를 함께 고쳐 서로 어긋나지 않게 한다. `updated`/날짜도 갱신.

| 위치 | 역할 | 갱신 시점 |
|------|------|-----------|
| `CLAUDE.md` | 매 세션 읽히는 핵심 카드 (이식 가능한 최신 상태) | 상태 변화 시 항상 |
| `docs/data.json` | 가족 페이지에 표시되는 데이터 | 무게·사료·다음일정·훈련단계 등 변화 시 |
| 해당 기록 파일 | 추이/이력 보존 | 항목별로 (아래) |

기록 파일 매핑:
- 몸무게 → `health/weight-log.md` (표에 행 추가)
- 접종·구충·병원 → `medical/schedule.md`
- 일상 메모 → `journal/YYYY-MM.md` (그 달 파일, 없으면 생성)
- 훈련 수행/회고 → `training/log.md`, 단계 변화는 `training/roadmap.md` + `docs/training.json`
- 사료·급식 변화 → `CLAUDE.md` 급여 섹션 + `docs/data.json`(feeding/feedingSchedule/mealBreakdown)

### 3. 메모리 갱신 (이 PC 로컬)
파일 기반 메모리 디렉터리(시스템 컨텍스트에 주어진 경로;
이 PC에서는 `C:\Users\winsu\.claude\projects\F--workspace-for-bbosyong\memory\`)에서:
- 새 durable 사실을 **한 파일 = 한 사실**로 저장. 같은 주제는 **기존 파일을 갱신**(중복 생성 금지). 틀린 건 삭제.
- frontmatter `type`: `user`/`feedback`/`project`/`reference` 적절히. feedback/project는 **Why/How to apply** 포함.
- 관련 메모리는 `[[name]]`로 링크.
- **`MEMORY.md` 인덱스에 한 줄 포인터**를 추가/수정 (한 줄 요약).

> 참고: 메모리는 이 PC에만 있다. **클라우드/모바일 세션에서는 메모리가 없을 수 있으니**,
> 이식 가능한 최신 상태는 반드시 `CLAUDE.md`(저장소 안)에 담아 둘 것. 메모리 단계는 가능할 때만 수행.

### 4. commit & push
변경을 의미 있는 단위로 커밋하고 **PR 없이 `main`에 직접 push**한다(사용자 승인됨).
커밋 메시지는 한국어로 무엇이 왜 바뀌었는지 간결히. 푸시하면 가족 페이지/모바일에 반영된다.

### 5. 마무리 보고
사용자에게 **무엇을 어디에 남겼는지** 짧게 보고한다. 예:
- 갱신한 메모리(새로 추가/수정)
- 동기화한 파일(CLAUDE.md / data.json / 기록 파일)
- push 커밋
- 다음에 챙길 것(예: 다가오는 접종 D-day)

## 페이지를 바꿨다면
`docs/` HTML/JSON을 고쳤으면 가능한 경우 로컬 서버(`python -m http.server`)+브라우저로
렌더링을 한 번 확인하고, JSON은 UTF-8로 파싱되는지 검증한 뒤 임시 파일을 정리한다.

## 하지 말 것
- 자명하거나 일회성인 내용을 메모리에 남기기
- 한 곳만 고치고 다른 동기화 위치를 빠뜨리기 (상태가 어긋남)
- 사용자 승인 없이 외부로 무언가를 새로 게시하기 (기존 push 규칙 범위 내에서만)
